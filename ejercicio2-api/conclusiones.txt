CONCLUSIONES Y HALLAZGOS

1. Los servicios de Signup y Login siempre retornan código HTTP 200,
   incluso cuando ocurre un error funcional.

2. Los errores se identifican únicamente en el cuerpo de la respuesta
   mediante el campo "errorMessage".

3. No existe diferenciación de códigos HTTP para errores como:
   - Usuario ya existente
   - Password incorrecto

4. Esto puede representar un riesgo desde el punto de vista de buenas
   prácticas REST, ya que no se usan códigos HTTP como 400 o 401.

5. Karate DSL permitió una implementación clara, legible y rápida
   de las pruebas de servicios REST.
